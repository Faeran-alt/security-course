Можно ли прочитать исходные данные из зашифрованного файла:
Нет. Файл содержит только шифртекст в формате Base64. Исходные данные полностью скрыты криптографически.
Что произойдёт если ввести неверный пароль при расшифровке:
GPG выдаст ошибку gpg: decryption failed: Bad session key или gpg: decryption failed: No secret key, и данные не будут расшифрованы.
Почему нельзя расшифровать файл зашифрованный для другого пользователя:
Файл зашифрован публичным ключом получателя. Расшифровать можно только соответствующим приватным ключом, который есть только у получателя. У тебя нет приватного ключа соседа — значит, расшифровать невозможно.
3. Цифровая подпись
Вывод gpg --verify (Good signature):
plain

gpg: Signature made Fri Jun 13 18:27:12 2026 CST
gpg:                using RSA key B5B69D174E47D1F3CF388E688FFCC344DBBB86F2
gpg: Good signature from "Faeran <faeranvoild@gmail.com>" [ultimate]

Вывод gpg --verify после изменения файла (BAD signature):
plain

gpg: Signature made Fri Jun 13 18:27:12 2026 CST
gpg:                using RSA key B5B69D174E47D1F3CF388E688FFCC344DBBB86F2
gpg: BAD signature from "Faeran <faeranvoild@gmail.com>" [ultimate]

Почему подпись перестала быть действительной после изменения файла:
Подпись содержит зашифрованный хеш (дайджест) оригинального файла. При любом изменении файла хеш изменяется и не совпадает с хешем в подписи → подпись становится невалидной (BAD signature).
Что именно проверяет gpg --verify:

    Целостность — файл не был изменён после подписания
    Аутентичность — подпись создана владельцем соответствующего приватного ключа
    Неотказуемость — подписавший не может отрицать факт подписи

5. rm vs shred
Table
Операция	Что происходит с данными на диске	Можно восстановить?
rm	Удаляется только запись в метаданных (inode), сами блоки данных остаются	Да, пока блоки не перезаписаны
shred	Содержимое блоков перезаписывается случайными данными несколько раз	Практически нет
Сценарий когда нужно затирать свободное место:
При продаже или утилизации жёсткого диска, когда ранее удалённые через rm файлы могут содержать конфиденциальные данные. Также при сдаче ноутбука в ремонт или при увольнении сотрудника, который работал с секретами.
